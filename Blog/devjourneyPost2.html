So you want to be a software developer?

My advice on how to get better and retain what you learn

Posted by Carl Hamilton on September 4th, 2017

Practice and make your own projects

Many developers make the mistake of just going through a learning process, this can be either through videos on Youtube or online tutorials or one of the many available books available out there.

I would say that is a BIG mistake! What will happen is you will learn small things that generally has repetition of the same concepts, however anything complex will be lost. A great way to learn would be to learn a concept or framework and then take the time to make some personal projects on the subject.

When you have made some projects you should then perform tests to make sure that you fully understand the concepts.

Recently I have been taking the concepts that i'm becoming familiar with to make some simple games such as Tic Tac Toe clone and a Calculator app.

Make notes and personalised cheat sheets

Everyone forgets things so make great use of Google as developers don't be ashamed to need to Google. There are so many great resources out there I use a Macbook Pro to do most of my developement and heavily make use of the Notes app, I believe it is essentail to keep some notes.

Explore APIs

An excellent learning resource is to study APIs, making use of an API could make all the difference to the level of experience you develop as a programmer.

API stands for Application Programming Interface see Wikipedia.org

Resources such as ProgrammableWeb which is a massive library for APIs that can be used.

Once your skills are finely tuned and you can produce APIs yourself it is possible to charge for them as they can become essential to Web applications, so don't be ashamed for trying to make some money while your on your journey.

I advise once you are done learning something that you feel even a little bit confident in then you should blog about it. No matter how much you know there is someone out there can benefit in your experience. Any learning is a good thing! Make use of Social Media as well, this can be a great help. Either Twitter or Facebook groups, don't sell yourself short.

Finally if you spend quite a bit of time trying to find a solution to a problem you experience, write a blog post about it! You do not need to only post positives or advice post your mishaps or stupid things you do I will be. :-)

Any code of your own that you haven't looked at for six months or more might as well have been written by someone else
