Learnt some concepts? now put them to use!

I decided to make a Calculator and a clone of an old classic game using C#

Posted by Carl Hamilton on September 11th, 2017

I wanted to make a game that was simple but used what I have learnt

So the other day I was contemplating what to make, I knew I wanted to take the concepts i've learnt so far with C# and wanted to make something simple but yet I didn't want to make it too simple as I was starting to understand some advanced concepts.

So I decided upon Tic Tac Toe. The finished application can be shown in the screenshot below.

The game can currently be played by two players, however as my coding skills get better I will add AI into it. If you are unaware TicTacToe is basically Naughts and crosses.

The idea was to make this ready available to run on Windows so I used WPF for this project.

The project can be viewed and downloaded from here

The code behind the project was built using C# and I made sure I created seperate methods and a Class, the idea behind this was to make it easier to read and be helpful if a third party was to read the code.

You should always code asking yourself could you read it if someone else had made it?

Next came the Calculator, simple but coded to be more complex for understanding

So with the Calculator I didn't want to just simply make a GUI version of a Calculator I coded in college.

I remember starting out with C++ and our class were given the task to code a Calculator using a console and back then the concept was fairly simple.

So a menu system was built asking the user with equation they want to use such as * / - and + then this called a switch statement and calculated the sum based on a case true statement.

This isn't very impressive now I look back at it, however it did get me a distinction and praise from the tutor due to my additional menu options such as calculating the binary and HEX. However I made this with a little more love which can be found here I have posted a screenshot below

The code behind this had to be a little more complicated than my college work, therefore I included enums and Classes and then used inheritence to call these. An example snippet can be seen below

One thing the Calculator had to do was accomadate deleting characters if you make a mistake and also prevent a repeating . should you make a mistake.

Seperate methods were used as a method or function should only complete one task, if you make this more complicated it as a program grows it could become very hard to refactor code at a later date.

Why this blog post?

I wanted to show a few examples to just let you know that no matter what pace you learn at there will be programs you can make, some useful for a long time and some you will hate as your experience grows. It is completely normal and you should always attempt to build things.

Don't like Tic Tac Toe? Then pick another one popular game used to clone back in the Turbo Pascal days was Pong and it is certainly a classic.

A Good programmer fixes a problem you do not know you have, in a way you do not understand
